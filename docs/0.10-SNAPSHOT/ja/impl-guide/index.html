本書は、本プロジェクトにおける実装作業を効率的かつ一貫性を持って進めるための指針を提供することを目的としています。

本書は、本プロジェクトに参画する開発者を対象としています。開発者が本書に示されるプロセス、及びアーキテクチャ記述に示される標準を遵守することにより、高品質なシステムを効率的に開発することを目指します。

ガイドの読み進め方

本書は以下の順序で読み進めることを推奨します。 この順序は、開発者が実際に動くアプリケーションとソースコードを手元で参照しながら、本書、及びアーキテクチャ記述の要点を確認できるよう構成されています。

  1. 環境構築
    開発環境構築手順に従い、開発に使用するPC上に実行・開発環境を構築します。

  2. アプリケーションの確認
    構築した環境でアプリケーションを実行します。 アプリケーションを操作し、実装されている機能を確認します。

  3. アプリケーションアーキテクチャの確認

    1. 論理的な構成
      アーキテクチャ記述のアプリケーション構成、コンポーネント構成を参照し、アプリケーションを構成するコンポーネントとその責務を確認します。

    2. 物理的な実装
      実装手順(自動生成)とそこで説明されている各コンポーネントのソースコードを参照し、コンポーネントの実装方法を確認します。

開発環境構築手順

必要なソフトウェアのインストール

開発には以下のソフトウェアが必要です。

  • Docker Desktop

  • JDK v21

  • Node.js v22

  • pnpm

  • Git

  • Visual Studio Code

次のOSごとの手順に従ってソフトウェアをインストールしてください。

Windows

WindowsではChocolateyを使用してソフトウェアをインストールします。

管理者権限でPowershellを起動します。(起動操作は下記参照)

  1. ショートカットキー Windows+R を実行します。

  2. 表示されたダイアログで powershell と入力します。

  3. ショートカットキー Ctrl+Shift+Enter を実行します。

表示されたPowershellウィンドウで以下のコマンドを実行します。

Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

続けて以下のコマンドを実行し、必要なソフトウェアをインストールします。

choco install -y docker-desktop
choco install -y corretto21jdk
choco install -y nodejs-lts
choco install -y pnpm
choco install -y git
choco install -y vscode

macOS

macOSではHomebrewを使用してソフトウェアをインストールします。

ターミナルで以下のコマンドを実行し、Homebrewをインストールします。

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

続けて以下のコマンドを実行し、必要なソフトウェアをインストールします。

brew install --cask docker
brew install --cask corretto@21
brew install node@22
brew install pnpm
brew install git
brew install --cask visual-studio-code

プロジェクト資材の取得とセットアップ

コマンドプロンプト(Windows)/ターミナル(macOS)で以下のコマンドを実行し、プロジェクト資材の取得と開発環境のセットアップを行います。

(1)
git clone https://github.com/project-au-lait/svqk.git
cd svqk

(2)
chcp 65001
mvnw install -T 1C -P setup,browse-e2etest

(3)
./mvnw install -T 1C -P setup,browse-e2etest
1 プロジェクト資材の取得
2 開発環境のセットアップ (コマンドプロンプト(Windows)の場合)
chcp 65001 はコマンドプロンプトの文字コードをUTF-8に設定するコマンド
3 開発環境のセットアップ (ターミナル(macOS)の場合)

上記のセットアップコマンドの処理内容、及び実現方式はビルド構造を参照してください。

登録画面

ここでは登録画面の実装手順を説明します。 説明ではリファレンス実装のチケット登録画面を例に挙げます。

Figure 1. 登録画面の例　チケット登録画面

登録画面には、登録内容を入力する項目と、登録を実行するボタンが配置されます。 また、登録画面では以下の2つの処理があります。

  1. 画面に遷移して来た際の初期化処理

  2. 画面に入力されたデータの登録処理

以降では、それぞれの処理に対する実装を説明します。

初期化処理

初期化処理は、当該画面への遷移イベントが発生してから画面が表示されるまでに実行される処理です。 登録画面の初期化処理では以下の処理が実行されます。

  • 入力項目の初期化
    画面表示時の入力項目の値を設定します。

  • 入力項目のバリデーションの設定
    入力項目にクライアントサイドで行う単項目チェックを設定します。

  • 画面要素の描画

登録画面の初期化処理のシーケンスは以下の通りです。

Table 3. 画面初期表示時の処理シーケンス

  1. ユーザーが登録画面に遷移します。

  2. PageLoaderのload functionが呼び出されます。

  3. PageLoaderが画面の入力値を格納するmodelのインスタンスを生成します。インスタンスのプロパティには画面要素の初期値を設定します。

  4. PageがPageLoaderから取得したmodelを元に画面のHTMLを構築します。また入力項目のバリデーションを設定します。

登録処理

登録画面の登録時の処理シーケンスは以下の通りです。

  1. ユーザーが登録操作を実行します。

  2. PageがWeb APIを呼び出します。

  3. ControlelrがPOSTされたdtoをentityに変換します。

  4. ControllerがServiceを呼び出します。

  5. ServiceがRepositoryを呼び出します。

  6. Repositoryがentityの状態に応じてINSERTまたはUPDATEを実行します。

  7. Repositoryがentityを返却します。

  8. Serviceがentityを返却します。

  9. Controllerがentityのidとversionプロパティを返却します。

マスターデータロード

ここではマスターデータロードの実装手順を説明します。 説明ではリファレンス実装のチケットステータスを例に挙げます。

  • マスターデータの取得
    画面で使用するマスターデータをBackendから取得します。

  • マスターデータの保持
    Backendから取得したマスターデータをstoreとして保持します。

マスターデータロード処理のシーケンスは以下の通りです。

Table 6. マスターデータロードの処理シーケンス

  1. ユーザーがFrontendにアクセスすると、初回アクセス時にLayoutLoaderのload functionが呼び出されます。

  2. LoadExecutorがMasterStoreのload処理を実行します。

  3. MasterStoreが画面で使用するコード値等を取得するWeb APIを呼び出します。

  4. ControllerがServiceを呼び出します。

  5. ServiceがRepositoryを呼び出します。

  6. RepositoryがDBに対しSELECTを実行します。

  7. ControllerがServiceから取得したentityをdtoに変換します。

  8. MasterStoreが取得した結果をstoreとして保持します。

ここで、LayoutLoaderはSvelteKitの layout.ts を指し、リファレンス実装ではsrc/route以下に配置されています。また、LoadExecutorはarchが提供する共通機能で、リファレンス実装ではsrc/lib/arch/masterに配置されています。

以降では、上記の処理シーケンスの各要素を以下の順に実装します。

  1. DB: マスターテーブルを作成とマスターデータのロードを行うマイグレーションスクリプトの実装

  2. Backend: マスターデータを取得するWeb APIの実装

  3. Frontend: Web API Clientの生成と組み込み
